# Fix: Erreur 404 avec Nginx et Flask

## Problème Identifié

Les URLs `/capture/login` et `/capture/` retournaient 404, alors que `/capture/health` fonctionnait.

### Cause

Lorsque `APPLICATION_ROOT = '/capture'` est défini dans Flask et que Nginx fait `proxy_pass http://capteur_app`, la requête complète (incluant `/capture`) est passée à Flask. Flask cherche alors `/capture/capture/login` au lieu de `/login`.

## Solution

### Approche : Nginx Strip le Préfixe

Nginx enlève le préfixe `/capture` avant de passer la requête à Flask.

**Modifications dans nginx.conf :**

```nginx
# AVANT (ne fonctionnait pas)
location /capture {
    proxy_pass http://capteur_app;
}

# APRÈS (fonctionne)
location /capture/ {         # ← Trailing slash important!
    proxy_pass http://capteur_app/;  # ← Trailing slash important!
}

# Redirection pour /capture sans slash
location = /capture {
    return 301 /capture/;
}
```

**Modifications dans app.py :**

```python
# AVANT (ne fonctionnait pas)
app.config['APPLICATION_ROOT'] = '/capture'

# APRÈS (fonctionne)
# Pas de APPLICATION_ROOT, Flask fonctionne à la racine
# Le cookie path reste /capture pour fonctionner correctement
app.config['SESSION_COOKIE_PATH'] = '/capture'
```

## Comment Ça Fonctionne Maintenant

### Flow de Requête

1. **Client → Nginx**
   ```
   GET /capture/login HTTP/1.1
   ```

2. **Nginx strip `/capture`**
   ```nginx
   location /capture/ {
       proxy_pass http://capteur_app/;
   }
   ```
   Avec les trailing slashes, Nginx enlève `/capture` et passe `/login`

3. **Nginx → Flask**
   ```
   GET /login HTTP/1.0
   X-Forwarded-Prefix: /capture
   ```

4. **Flask route la requête**
   ```python
   @app.route('/login')  # ← Match!
   def login():
       ...
   ```

5. **Flask → Nginx → Client**
   - Cookie défini sur path `/capture` (fonctionne)
   - Redirect vers `/` devient `/capture/` (grâce à ProxyFix)

### Mapping des URLs

| URL Client | Nginx reçoit | Flask reçoit | Route Flask |
|------------|--------------|--------------|-------------|
| `/capture/` | `/capture/` | `/` | `@app.route('/')` |
| `/capture/login` | `/capture/login` | `/login` | `@app.route('/login')` |
| `/capture/upload` | `/capture/upload` | `/upload` | `@app.route('/upload')` |
| `/capture/health` | `/capture/health` | `/health` | `@app.route('/health')` |

## Points Importants

### Trailing Slashes

Les trailing slashes dans nginx.conf sont **CRITIQUES** :

```nginx
# ✓ CORRECT
location /capture/ {
    proxy_pass http://capteur_app/;
}

# ✗ INCORRECT (ne strip pas le préfixe)
location /capture {
    proxy_pass http://capteur_app;
}
```

### Redirection /capture

Sans trailing slash, redirige vers la version avec slash :

```nginx
location = /capture {
    return 301 /capture/;
}
```

### Cookies

Le cookie path reste `/capture` pour que les sessions fonctionnent :

```python
app.config['SESSION_COOKIE_PATH'] = '/capture'
```

Ainsi le cookie n'est envoyé que pour les URLs commençant par `/capture/`.

### url_for() dans Flask

Grâce à `ProxyFix` et `X-Forwarded-Prefix`, Flask génère les bonnes URLs :

```python
url_for('login')  # Génère /capture/login (pas /login)
```

## Alternatives (Non Utilisées)

### Option 1: Blueprint avec Préfixe

```python
from flask import Blueprint

bp = Blueprint('main', __name__, url_prefix='/capture')

@bp.route('/login')
def login():
    ...

app.register_blueprint(bp)
```

Problème: Plus complexe, nécessite de modifier toutes les routes.

### Option 2: DispatcherMiddleware

```python
from werkzeug.middleware.dispatcher import DispatcherMiddleware

app = DispatcherMiddleware(None, {
    '/capture': app
})
```

Problème: Moins flexible, configuration complexe.

### Option 3: Garder APPLICATION_ROOT

Garder `APPLICATION_ROOT = '/capture'` et ne pas stripper dans Nginx.

Problème: Ne fonctionne pas bien avec Flask, nécessite des ajustements partout.

## Solution Actuelle

**Notre solution (la plus simple) :**
- Nginx strip le préfixe
- Flask fonctionne normalement à la racine
- Cookies configurés sur `/capture`
- ProxyFix gère les redirections

C'est la méthode standard recommandée pour Flask derrière un reverse proxy avec préfixe.

## Dépannage

### Toujours des 404

1. Vérifier les trailing slashes dans nginx.conf
2. Vérifier que APPLICATION_ROOT n'est pas défini dans app.py
3. Recharger Nginx: `sudo systemctl reload nginx`
4. Redémarrer l'app: `sudo systemctl restart capteur`

### Les redirections ne fonctionnent pas

Vérifier que ProxyFix est activé dans app.py:

```python
from werkzeug.middleware.proxy_fix import ProxyFix
app.wsgi_app = ProxyFix(app.wsgi_app, x_for=1, x_proto=1, x_host=1, x_prefix=1)
```

### Les cookies ne fonctionnent pas

Vérifier le cookie path:

```python
app.config['SESSION_COOKIE_PATH'] = '/capture'
```

Dans le navigateur (DevTools → Application → Cookies), vérifier :
- Name: `capteur_session`
- Path: `/capture`
- Domain: votre domaine
